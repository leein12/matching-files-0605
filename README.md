# 거래처 매칭 프로그램

## 프로그램 목적

엑셀 파일 A와 B의 거래처 정보(거래처명, 주소, 전화번호)를 정규화한 뒤 유사도를 계산하여, A의 각 행에 대해 B에서 유사한 후보를 찾아 결과 파일 C.xlsx를 생성합니다.

- 비교 대상은 **B.xlsx만** 사용합니다. (V.xlsx는 사용하지 않습니다.)
- 거래처키는 비교에 사용하지 않으며, 결과 표시용으로만 사용합니다.
- 80점 이상 후보만 결과에 포함되며, 최대 5개까지 표시합니다.


## 입력 파일 위치

- `input/A.xlsx`
- `input/B.xlsx`

필수 컬럼: 거래처키, 거래처명, 거래처주소, 전화번호

## 출력 파일 위치

- `output/C.xlsx`

## 점수 계산 방식

| 항목 | 만점 |
|------|------|
| 거래처명 | 50점 |
| 주소 | 50점 |
| 전화번호 가점 | 20점 |

**최종 점수** = `min(100, 거래처명점수 + 주소점수 + 전화번호가점)`

### 거래처명 (최대 50점)
- base_key 일치/포함 관계
- rapidfuzz 문자열 유사도
- 한글 초성 보정
- 보조 일치 보너스/일반명 패널티

### 주소 (최대 50점)
- 광역/도 일치 (8점)
- 시/군/구 일치 (12점)
- 읍/면/동 일치 (5점)
- 도로명 일치 (최대 15점)
- 건물번호 일치 (최대 10점)

### 전화번호 가점
- 정규화 후 완전 일치: +20점
- 마지막 8자리 일치: +10점
- 한쪽 비어 있거나 불일치: +0점

## 결과 포함 기준

- **80점 이상** 후보만 C.xlsx에 표시됩니다.
- 후보는 점수 내림차순으로 **최대 5개**까지 표시됩니다.
- 후보가 없으면 후보 영역은 빈 값으로 둡니다.

## 결과 파일에 표시하지 않는 정보

다음 정보는 내부 계산에만 사용되며 C.xlsx에 출력하지 않습니다.

- 정규화 정보 (거래처명, 주소, 시도, 시군구, 전화번호)
- 원본행번호
- 상세 점수 (거래처명점수, 주소점수, 전화번호가점)
- 판정, 검토사유, 표시용 컬럼

## 예외 주소 처리

특정 주소 쌍을 동일하게 취급하려면 `config.py`의 `EQUIV_MAP`에 추가하세요.

```python
EQUIV_MAP = {
    "부산 사하구 사하로186번가길 1": "부산 사하구 사하로 188",
    "부산 사하구 사하로 188": "부산 사하구 사하로186번가길 1",
}
```

## 주의사항

- V.xlsx는 사용하지 않습니다.
- 전화번호는 가점이며 필수 조건이 아닙니다. 전화번호가 없어도 이름+주소만으로 80점 이상이면 후보가 될 수 있습니다.
- 오프라인 환경에서 동작합니다. (외부 API 미사용)
