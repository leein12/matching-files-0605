# 거래처 매칭 프로그램

## 프로그램 목적

엑셀 파일 A와 B의 거래처 정보(거래처명, 주소, 전화번호)를 정규화한 뒤 유사도를 계산하여, A의 각 행에 대해 B에서 유사한 후보를 찾아 결과 파일 C.xlsx를 생성합니다.

- 비교 대상은 **B.xlsx만** 사용합니다. (V.xlsx는 사용하지 않습니다.)
- 거래처키는 비교에 사용하지 않으며, 결과 표시용으로만 사용합니다.
- 기본 설정에서는 계산된 후보를 점수와 무관하게 포함하며, 최대 3개까지 표시합니다.
- 최고점수 기준으로 확정/검토/미매칭을 분류하고, 요약/기준 시트를 함께 생성합니다.

## 실행 방법

```bash
python main.py
```

## 입력 파일 위치

- `input/A.xlsx`
- `input/B.xlsx`

기본 유사도 매칭 필수 컬럼: 거래처키, 거래처명, 거래처주소, 전화번호

고객명 정확일치 매핑 필수 컬럼: 고객코드, 고객명

## 실행 모드

`config.ini`의 `[matching] mode` 값으로 실행 방식을 선택할 수 있습니다.

- `auto`: A/B 양쪽에 고객코드, 고객명 컬럼이 있으면 고객명 정확일치 모드, 없으면 기존 유사도 매칭 모드로 실행합니다.
- `customer_exact`: 고객명 100% 일치 기준으로 B의 기존 거래처를 매핑합니다.
- `company_similarity`: 기존 거래처명 + 주소 + 전화번호 유사도 매칭으로 실행합니다.

## 출력 파일 위치

- `output/C.xlsx`

## 점수 계산 방식

| 항목 | 만점 |
|------|------|
| 거래처명 | 50점 |
| 주소 | 50점 |
| 전화번호 가점 | 20점 |

**최종 점수** = `min(100, 거래처명점수 + 주소점수 + 전화번호가점)`

### 거래처명 (최대 50점)
- base_key 일치/포함 관계
- rapidfuzz 문자열 유사도
- 한글 초성 보정
- 보조 일치 보너스/일반명 패널티

### 주소 (최대 50점)
- 광역/도 일치 (8점)
- 시/군/구 일치 (12점)
- 읍/면/동 일치 (5점)
- 도로명 일치 (최대 15점)
- 건물번호 일치 (최대 10점)

### 전화번호 가점
- 정규화 후 완전 일치: +20점
- 마지막 8자리 일치: +10점
- 한쪽 비어 있거나 불일치: +0점

## 결과 포함 기준

- 기본 `match_threshold`는 0점입니다.
- 후보는 점수 내림차순으로 **최대 3개**까지 표시됩니다.
- 후보가 없으면 후보 영역은 빈 값으로 둡니다.

## 매칭결과 구간과 요약

`매칭결과` 컬럼은 `최고점수` 기준으로 아래 기본 구간을 적용합니다.

- 확정: 80~100
- 검토: 70~79
- 미매칭: 0~69

결과 파일에는 `요약` 시트와 `기준` 시트가 함께 생성됩니다.

- `요약` 시트: 확정/검토/미매칭별 건수와 비중을 수식으로 집계합니다.
- `기준` 시트: 확정/검토/미매칭의 시작점수와 종료점수를 수정할 수 있습니다.
- 엑셀에서 `기준` 시트의 구간을 바꾸면 `매칭결과`와 `요약` 값이 다시 계산됩니다.

## 결과 파일에 표시하지 않는 정보

다음 정보는 내부 계산에만 사용되며 C.xlsx에 출력하지 않습니다.

- 정규화 정보 (거래처명, 주소, 시도, 시군구, 전화번호)
- 원본행번호
- 상세 점수 (거래처명점수, 주소점수, 전화번호가점)
- 판정, 검토사유, 표시용 컬럼

## 예외 주소 처리

특정 주소 쌍을 동일하게 취급하려면 `config.py`의 `EQUIV_MAP`에 추가하세요.

```python
EQUIV_MAP = {
    "부산 사하구 사하로186번가길 1": "부산 사하구 사하로 188",
    "부산 사하구 사하로 188": "부산 사하구 사하로186번가길 1",
}
```

## 주의사항

- V.xlsx는 사용하지 않습니다.
- 전화번호는 가점이며 필수 조건이 아닙니다.
- 오프라인 환경에서 동작합니다. (외부 API 미사용)
