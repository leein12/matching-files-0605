거래처 매칭 프로그램 사용 매뉴얼
================================

1. 구성 파일
-----------

이 폴더는 아래 구조로 사용합니다.

matching-files/
  matching-files.exe
  README.txt
  config.ini
  input/
    A.xlsx
    B.xlsx
  output/
    C.xlsx

사용자는 별도 Python 설치나 pip install 없이 matching-files.exe를 실행하면 됩니다.


2. 실행 방법
-----------

1) input 폴더에 A.xlsx와 B.xlsx를 넣습니다.
2) Windows 탐색기에서 matching-files.exe를 더블클릭합니다.
3) 처리 완료 후 output 폴더에 C.xlsx가 생성됩니다.

입력 파일명은 기본적으로 아래와 같습니다.

- input/A.xlsx
- input/B.xlsx

출력 파일명은 기본적으로 아래와 같습니다.

- output/C.xlsx


3. 입력 엑셀 필수 컬럼
---------------------

기존 거래처 유사도 매칭 모드에서는 A.xlsx와 B.xlsx에 아래 컬럼이 있어야 합니다.

- 거래처키
- 거래처명
- 거래처주소
- 전화번호

고객명 정확일치 매핑 모드에서는 A.xlsx와 B.xlsx에 아래 컬럼이 있어야 합니다.

- 고객코드
- 고객명

거래처키는 점수 계산에 사용하지 않고 결과 표시용으로만 사용합니다.
비교 대상 파일은 B.xlsx만 사용합니다. V.xlsx는 사용하지 않습니다.


4. 점수 계산 방식
----------------

최종 점수는 100점 만점입니다.

최종점수 = min(100, 거래처명점수 + 주소점수 + 전화번호가점)

- 거래처명점수: 최대 50점
- 주소점수: 최대 50점
- 전화번호가점: 최대 20점

기본 설정에서는 계산된 후보를 점수와 무관하게 C.xlsx에 표시합니다.
후보는 A 한 행당 최대 3개까지 표시합니다.
최고점수 기준으로 확정/검토/미매칭을 분류합니다.


5. 결과 파일 안내
----------------

C.xlsx에는 아래 정보만 표시합니다.

- A 원본 거래처키/거래처명/거래처주소/전화번호
- 매칭후보수
- 최고점수
- 매칭결과
- 후보1~후보3의 B 거래처키/거래처명/거래처주소/전화번호/총점

추가 시트:

- 요약: 확정/검토/미매칭별 건수와 비중을 수식으로 집계합니다.
- 기준: 확정/검토/미매칭의 시작점수와 종료점수를 수정할 수 있습니다.

엑셀에서 기준 시트의 구간을 변경하면 매칭결과와 요약 값이 다시 계산됩니다.

아래 정보는 내부 계산에만 사용하며 결과 파일에는 표시하지 않습니다.

- 정규화 거래처명
- 정규화 주소
- 정규화 시도/시군구
- 정규화 전화번호
- 원본행번호
- 거래처명점수
- 주소점수
- 전화번호가점
- 판정
- 검토사유
- 표시용 컬럼


6. config.ini 사용법
-------------------

config.ini는 메모장 등 txt 에디터로 수정할 수 있습니다.
수정 후 matching-files.exe를 다시 실행하면 변경값이 적용됩니다.

[paths]

input_dir = input
output_dir = output
a_file = A.xlsx
b_file = B.xlsx
output_file = C.xlsx

- input_dir: 입력 폴더명
- output_dir: 출력 폴더명
- a_file: A 파일명
- b_file: B 파일명
- output_file: 결과 파일명

예: 입력 파일명을 A_거래처.xlsx로 바꾸고 싶다면

a_file = A_거래처.xlsx


[matching]

mode = auto
match_threshold = 0
max_candidates = 3
pause_on_exit = true

[result_ranges]

확정 = 80,100
검토 = 70,79
미매칭 = 0,69

- mode: 실행 모드입니다.
  auto: A/B 양쪽에 고객코드, 고객명 컬럼이 있으면 고객명 정확일치 모드로 실행하고,
        없으면 기존 거래처 유사도 매칭 모드로 실행합니다.
  customer_exact: 고객명 100% 일치 기준으로 B의 기존 거래처를 매핑합니다.
  company_similarity: 기존 거래처명+주소+전화번호 유사도 매칭으로 실행합니다.

- match_threshold: 후보 표시 최소 총점입니다.
  기본값 0은 계산된 후보를 점수와 무관하게 표시합니다.
  customer_exact 모드에서는 사용하지 않습니다.

- max_candidates: A 한 행당 표시할 최대 후보 수입니다.
  기본값은 3입니다.

- pause_on_exit: 탐색기 실행 시 종료 전에 Enter 입력을 기다릴지 여부입니다.
  true 또는 false를 사용할 수 있습니다.

- result_ranges: 매칭결과 구간입니다.
  형식은 "구분 = 시작점수,종료점수"입니다.
  기본값은 확정 80~100, 검토 70~79, 미매칭 0~69입니다.


[name]

general_name_risk_tokens = 서울,중앙,제일,성모,우리,하나,행복,사랑

- 일반명이어서 충돌 위험이 있는 거래처명 토큰입니다.
- 쉼표로 구분해서 추가/삭제할 수 있습니다.


[admin_alias]

인천|남구 = 인천|미추홀구
경북|군위군 = 대구|군위군

- 행정구역 명칭 변경 또는 별칭을 지정합니다.
- 형식은 아래와 같습니다.

원본시도|원본시군구 = 정규화시도|정규화시군구


[equiv_map]

- 서로 같은 주소로 강제 처리할 예외 주소를 지정합니다.
- 형식은 아래와 같습니다.

주소1 = 주소2

예:

부산 사하구 사하로186번가길 1 = 부산 사하구 사하로 188
부산 사하구 사하로 188 = 부산 사하구 사하로186번가길 1


7. 오류 처리
-----------

- input 폴더가 없으면 자동 생성 후 안내 메시지를 표시합니다.
- A.xlsx 또는 B.xlsx가 없으면 오류 메시지를 표시합니다.
- 필수 컬럼이 없으면 어떤 컬럼이 없는지 표시합니다.
- C.xlsx가 엑셀에서 열려 있으면 저장 실패할 수 있습니다. 파일을 닫고 다시 실행하세요.


8. 권장 사용 절차
----------------

1) 기존 output/C.xlsx가 열려 있다면 닫습니다.
2) input/A.xlsx, input/B.xlsx를 최신 파일로 교체합니다.
3) matching-files.exe를 실행합니다.
4) output/C.xlsx를 확인합니다.
